5.1.1 Why Computers Are Connected in a Network
‚ñº
SUBJECT VOCABULARY

Network ‚Äî two or more computers connected together so they can communicate and share resources.

Server ‚Äî a powerful computer that provides services (file storage, printing, email, web pages) to other computers on a network. A small network may have one server; a large business may have hundreds.

Network service ‚Äî an application running on a server that provides facilities or operations to users, such as data storage, printing, or communications.

Connecting computers in a network gives huge advantages over standalone (single) computers:

1
Share resources ‚Äî printers, scanners and internet connections can be shared between all users on the network, reducing cost (e.g. one printer for 30 computers).
2
Share files & software ‚Äî data and programs stored centrally can be accessed from any machine on the network. No need for individual copies on every PC.
3
Communication ‚Äî users can send emails, messages and share documents with others on the same network instantly.
4
Centralised management ‚Äî an administrator can install software, apply updates and manage user accounts from a single server, saving huge amounts of time.
5
Centralised backup ‚Äî data from all machines is backed up automatically from the server, reducing risk of data loss.
6
Security management ‚Äî user logins, access controls and monitoring can all be managed centrally by one administrator.
e.g. In a school, students log in from any computer and access their files ‚Äî the files are on the server, not that specific machine. One printer serves the whole office. Software updates are pushed to every machine at once.
‚úèÔ∏è Exam tip: If asked to give advantages of networking, always mention sharing resources AND centralised management/backup. Two separate points = two marks. Don't just say "share things" ‚Äî be specific! Internet-connected fridges are a real example of networked devices ‚Äî any smart device counts!
5.1.2 Types of Networks & Usage Models
‚ñº
SUBJECT VOCABULARY

Wireless LAN (WLAN) ‚Äî a LAN where connected devices use high-frequency radio waves rather than cables. A wireless router allows all household devices to access the internet and share resources.

Wide Area Network (WAN) ‚Äî connects two or more LANs across a large geographical area, usually under shared ownership (e.g. leased telecom lines). The Internet is the largest WAN.

Personal Area Network (PAN) ‚Äî short-range network for personal devices. Uses Bluetooth (up to ~10m) or USB.

5.1.4 Network Data Speeds, Bandwidth & Latency
‚ñº
SUBJECT VOCABULARY

Bandwidth ‚Äî the maximum amount of data that can be transmitted over a network connection in a given time. Usually measured in bits per second (bps). Higher bandwidth = more data can travel at once. Think of it like the width of a motorway ‚Äî more lanes = more cars at once.

Latency ‚Äî the delay between a signal being sent and being received. Measured in milliseconds (ms). Low latency is critical for gaming, video calls and real-time control. A high-latency connection causes lag even if bandwidth is high.

Throughput ‚Äî the actual amount of data successfully transmitted per second in practice ‚Äî often lower than the theoretical maximum bandwidth due to network congestion, packet loss and overhead.

üì° Bandwidth ‚Äî Think "Width of Pipe"

A 10 Gbps fibre link has much more bandwidth than a 10 Mbps copper link ‚Äî it can carry far more data simultaneously.

Bandwidth is shared among all users on a network segment. More users = less bandwidth per person ‚Üí slower browsing.
‚è±Ô∏è Latency ‚Äî Think "Reaction Time"

Latency is caused by: propagation delay (distance the signal travels), processing time in routers/switches, and queuing delays under load.

A low-bandwidth link with low latency can feel faster for interactive tasks than a high-bandwidth but high-latency link.

Network data speeds are measured in bits per second (bps). Higher = faster.

5.1.5 Network Protocols
‚ñº
SUBJECT VOCABULARY

Protocol ‚Äî an agreed set of rules that define how devices communicate on a network. Both sender and receiver must follow the same protocol.

Without protocols, devices from different manufacturers couldn't communicate. You need to know these eight protocols:

E
Ethernet ‚Äî the standard wired LAN protocol. Defines how data is transmitted over physical cables using MAC addresses. Most common for home/office LANs.
W
Wi-Fi ‚Äî the standard wireless LAN protocol. Uses radio waves to transmit data without cables. Based on IEEE 802.11 standards.
T
TCP/IP ‚Äî the foundation of the Internet. TCP breaks data into packets and ensures delivery. IP addresses and routes each packet. Without this, the Internet doesn't work.
H
HTTP ‚Äî HyperText Transfer Protocol. Used by web browsers to request and receive web pages from servers. Unsecured.
S
HTTPS ‚Äî HTTP Secure. The encrypted version of HTTP. Uses TLS encryption to protect data in transit. Essential for online shopping, banking. Look for üîí in browser.
F
FTP ‚Äî File Transfer Protocol. Used to upload and download files between a client and a server. e.g. uploading a website to a web host.
P
POP3 ‚Äî Post Office Protocol 3. Downloads emails from server to device and deletes from server. Good for one device only.
S
SMTP ‚Äî Simple Mail Transfer Protocol. Used to send emails (and between mail servers).
I
IMAP ‚Äî Internet Message Access Protocol. Stores emails on the server and syncs across all your devices. Better than POP3 for multi-device use.

How Packet Switching works: A large message (e.g. a video) is split into many packets. Each packet is sent independently ‚Äî they may take different routes through the internet and arrive out of order. 5.1.7 Network Topologies
‚ñº
SUBJECT VOCABULARY

Network topology ‚Äî the physical or logical arrangement of devices and connections in a network.

Fault tolerant ‚Äî a network (or system) that continues to operate correctly even when one or more components fail. A mesh topology is highly fault tolerant because data can be rerouted via an alternative path if a link fails. Fault tolerance is a key design goal for critical networks such as hospitals, military systems and the Internet.

‚≠ê Star Topology ‚Äî Most Common
All devices connect to a central switch. Data goes device ‚Üí switch ‚Üí destination device.

‚úÖ Fast, easy to add devices, fault in one cable doesn't affect others
‚ùå If the switch fails, whole network fails. Expensive (cable per device)
‚ö†Ô∏è Not as fault tolerant as mesh ‚Äî switch is a single point of failure
üöå Bus Topology
All devices share a single backbone cable. Data travels both directions and all devices see it ‚Äî only the addressed device keeps it.

‚úÖ Cheap & easy to set up, less cable
‚ùå Cable break = whole network fails. Slow when busy (data collisions)
‚ö†Ô∏è Not fault tolerant ‚Äî a single cable failure stops all devices
üíç Ring Topology
Devices connected in a closed loop. Data travels in one direction around the ring, passing through each device until it reaches the destination.

‚úÖ Data flows in one direction ‚Äî no collisions. Equal access for all devices
‚ùå One device/cable fails = whole ring breaks. Slow (data must pass every device)
‚ö†Ô∏è Not fault tolerant ‚Äî single failure breaks the entire ring
üï∏Ô∏è Mesh Topology ‚Äî Most Fault Tolerant
Every device connects to every other device. Data can take multiple routes ‚Äî if one fails, another is used automatically.

‚úÖ Highly fault tolerant ‚Äî no single point of failure. If one link breaks, data takes an alternative route automatically
‚úÖ Multiple redundant paths mean the network keeps working even when components fail
‚ùå Very expensive ‚Äî lots of cable required. Complex to set up and manage
üï∏Ô∏è Used in: The Internet (partial mesh), military networks, hospital critical systems, and IoT sensor networks ‚Äî anywhere where uptime and fault tolerance are essential.
‚úèÔ∏è Exam tip: Star is by far the most commonly asked. Know the switch is the central device. For every topology, know one advantage AND one disadvantage. Mesh = fault tolerant is a key examiner keyword ‚Äî use it! The Internet uses a partial mesh. Most modern LANs use star.
üõ°Ô∏è Fault Tolerance ‚Äî Which Topology Wins?
Fault tolerant = the network continues working when a device or connection fails.

ü•á Mesh ‚Äî highest fault tolerance. Multiple paths exist, so data is automatically rerouted around failures. Used in the Internet and critical infrastructure.
ü•à Star ‚Äî single cable failure only affects one device (good), BUT if the switch fails, all devices lose connection (bad).
ü•â Ring ‚Äî NOT fault tolerant. One failure breaks the entire ring.
‚ùå Bus ‚Äî NOT fault tolerant. One cable break stops all devices.

5.1.8 Mobile Communication Standards
‚ñº
SUBJECT VOCABULARY

Bandwidth (mobile) ‚Äî the range of frequencies available for transmitting data in a mobile network. Each generation uses wider frequency bands, allowing more data to be transmitted simultaneously ‚Äî this is why each generation feels faster.

Latency (mobile) ‚Äî the delay from when data is sent to when it is received. Critical for real-time applications such as video calls, online gaming, and remote surgery. 5G achieves latency as low as 1ms, compared to ~50ms for 4G.

Base station (transceiver) ‚Äî a fixed transmitter/receiver that provides wireless coverage for a cell. Each base station covers an area of 9‚Äì21 miles.

Handover ‚Äî the seamless transfer of a mobile connection from one base station to another as the user moves between cells. The user is not notified; it happens automatically.

GSM (Global System for Mobile Communications) ‚Äî the standard for mobile communications, developed by the European Telecommunications Standards Institute.

üì° How Cellular Networks Work: A mobile network is distributed through cells ‚Äî each cell has a fixed location base station (transceiver) that provides wireless coverage. When a user moves out of one cell's range, control is automatically transferred to the next base station ‚Äî this is called handover. The subscriber is not notified. The standard for mobile communications is GSM, developed by the European Telecommunications Standards Institute.
